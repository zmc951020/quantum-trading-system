
# 开源参考项目研究总结

## 1. langchain-trading-agents (最有参考价值！)

**项目地址**：https://github.com/aitrados/langchain-trading-agents

**核心特点**：
- 基于LangChain的完整多智能体交易系统
- 6个专业角色的智能体分工协作
- 支持Ollama本地模型
- 支持MCP (Model Context Protocol)
- 支持多LLM提供商：OpenAI、Ollama、Deepseek、Gemini、Anthropic、Qwen等

---

### 1.1 6个智能体角色

| 角色 | 名称 | 职责 |
|------|------|------|
| 🎩 经理分析师 | Benjamin Frost | 总体协调和战略规划，任务分派给其他5人，控制流程和输出格式，汇总优先级判断 |
| 📊 技术指标分析师 | Alexander III | 基于传统技术指标做量化分析（MA、RSI、MACD、布林等），输出指标序列和信号 |
| 📈 价格行为分析师 | Edward Sterling | 识别价格行为、形态和关键支撑/阻力（蜡烛图模式、趋势线、结构性突破） |
| 📰 新闻分析师 | QIANG WANG | 抓取并分析新闻与情绪，评估短期影响并给出调整建议 |
| 📅 事件分析师 | Harrison Black | 结合经济日历和事件影响（利率决议、数据发布），判断事件窗口内的策略调整 |
| 🎯 决策者 | MR. Nightingale | 根据前五人输出做最终交易决策或建议（进场、出场、仓位和风险规则） |

### 1.4 使用示例

#### 单个智能体调用
```python
from common_lib_example import *
model_config = get_llm_model_config(ModelProvider.OLLAMA)

async def main():
    indicator_analyst_llm = PriceActionAnalyst(**model_config)
    result = await indicator_analyst_llm.analyze(query)
    print(result)
```

#### 多智能体协作 (AI Bus)
```python
from langchain_trading_agents.bus_controls.ai_bus_control import AiBusControl, GraphState
from langchain_trading_agents.llm_model.sub_agents import ManagerAnalyst, DecisionMakerAnalyst

manager_ai = AiBusControl(ManagerAnalyst(**model_config), DecisionMakerAnalyst(**model_config))
manager_ai.add_sub_agent(
    IndicatorAnalyst(**model_config),
    PriceActionAnalyst(**model_config),
    NewsAnalyst(**model_config),
    EventAnalyst(**model_config),
)
result: GraphState = await manager_ai.a_analyze(ask)
```

## 4. 技术栈建议

### 4.1 框架选择
- **强烈推荐 LangChain** —— langchain-trading-agents已经验证了可行性
- 支持Ollama本地模型，正好我们有qwen2.5-coder
- 完整的工具调用机制

### 4.2 实现顺序
1. **第一阶段**：实现基础的智能代理框架 (借鉴AiBusControl)
2. **第二阶段**：实现单个智能体 (策略代理 + 系统健康检查代理)
3. **第三阶段**：多智能体协作
4. **第四阶段**：UI集成
