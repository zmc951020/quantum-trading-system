
# 智能克隆技能研究报告

## 摘要

本文档研究了GitHub上三个最先进的AI自我进化项目，为我们的量化系统智能机器人提供技术参考和设计灵感。

---

## 一、发现的核心项目

### 1.1 Self-Improving Agent (自我进化代理)

**GitHub地址**: https://github.com/peterskoett/self-improving-agent

**项目类型**: OpenClaw Skill (技能插件)

**核心特点**:
- **Instinct机制**: 捕获原子级行为模式
- **Confidence评分**: 每个学习条目都有置信度
- **Project隔离**: 按项目隔离知识
- **Promote机制**: 从项目级升格到全局级

**工作流程**:
```
观察 → 生成Instinct → 验证 → 提炼为Skill
```

**数据结构**:
```
homunculus/
├── identity.json           # 身份配置
├── projects.json           # 项目映射
├── instincts/
│   ├── personal/          # 全局本能
│   └── project-{hash}/    # 项目特定
└── .learnings/            # 学习记录
```

**可借鉴点**:
1. **Pre/Post工具钩子**: 在每次工具调用前后监控
2. **置信度阈值**: 只有高置信度(&gt;0.7)才自动应用
3. **项目隔离**: 不同项目知识隔离，避免污染
4. **升级机制**: 多个项目验证后升级为全局技能

---

### 1.2 GenericAgent (通用自我进化代理)

**GitHub地址**: https://github.com/lsdefine/GenericAgent

**项目类型**: 完整Agent框架 (Python)

**核心特点**:
- **3K行核心代码**: 极简架构
- **9个原子工具**: 基础工具集
- **技能结晶机制**: 任务路径自动固化
- **5层记忆系统**: L0-L4分层记忆
- **6倍Token效率**: 相比通用框架

**5层记忆系统**:
| 层级 | 名称 | 作用 |
|------|------|------|
| L0 | Meta Rules | 基础行为约束（宪法） |
| L1 | Insight Index | 快速路由索引 |
| L2 | Global Facts | 长期稳定知识 |
| L3 | Task Skills/SOPs | 可复用任务流程 |
| L4 | Session Archive | 会话归档记录 |

**进化循环**:
```
新任务 → 自主探索 → 执行路径固化为Skill → 写入L3 → 下次直接调用
```

**可借鉴点**:
1. **分层记忆**: 不同类型知识分层存储
2. **技能结晶**: 成功路径自动保存
3. **Token优化**: 信息密度最大化
4. **自举能力**: 项目由自己完成

---

### 1.3 Evolver (基因组进化引擎)

**GitHub地址**: https://github.com/EvoMap/evolver

**项目类型**: 进化引擎 (Node.js)

**核心设计**: GEP (Genome Evolution Protocol)

**GEP协议三大概念**:
1. **Gene (基因)**: 最小进化单元
   ```json
   {
     "id": "repair_date_parsing",
     "intent": "修复日期解析",
     "prompt": "改进日期处理...",
     "validation": ["tests/...", "commands/..."],
     "signals": ["日志错误模式"]
   }
   ```

2. **Capsule (胶囊)**: 基因套餐，解决复合问题
3. **EvolutionEvent (进化事件)**: 完整记录每次进化

**四种策略模式**:
| 模式 | 比例 | 说明 |
|------|------|------|
| balanced | 50%创新,30%优化,20%修复 | 日常运行 |
| innovate | 80%创新 | 稳定期加速探索 |
| harden | 40%修复 | 大改动后收敛 |
| repair-only | 100%修复 | 紧急修复模式 |

**可借鉴点**:
1. **GEP协议**: 结构化进化规范
2. **审计记录**: 每次进化都可追溯
3. **策略模式**: 不同阶段不同进化策略
4. **验证流程**: 进化后必须验证

### 2.2 核心组件设计

#### 组件1: InstinctRecorder (本能记录器)

**借鉴**: Self-Improving Agent

**功能**:
- 监控所有工具调用
- 记录成功/失败
- 标注置信度
- 按项目隔离

**数据结构**:
```json
{
  "instinct_id": "uuid",
  "project_hash": "项目标识",
  "pattern": "检测到的模式",
  "action": "成功的行动",
  "confidence": 0.85,
  "created_at": "时间戳",
  "verification_count": 3
}
```

---

#### 组件2: SkillCrystallizer (技能结晶器)

**借鉴**: GenericAgent

**功能**:
- 观察任务执行路径
- 抽象可复用流程
- 生成标准操作程序SOP
- 存入L3记忆层

**结晶触发条件**:
1. 同一模式成功执行 ≥3次
2. 用户明确标注"保存为技能"
3. 置信度 ≥0.85

---

#### 组件3: EvolverEngine (进化引擎)

**借鉴**: Evolver (GEP)

**功能**:
- 扫描学习信号
- 匹配基因库
- 生成进化方案
- 执行验证流程
- 记录进化事件

**进化基因库示例**:
| 基因ID | 意图 | 触发信号 |
|--------|------|----------|
| gene_001 | 优化参数读取 | 策略加载超时 |
| gene_002 | 增强风控提示 | 风控漏检次数上升 |
| gene_003 | 优化器参数调优 | 优化收敛过慢 |
| gene_004 | 回测报告增强 | 用户看不懂回测结果 |

## 四、关键设计决策

### 4.1 本地优先原则

- **原因**: DeepSeek API成本高，隐私敏感
- **方案**: 优先使用我们的qwen2.5-coder本地模型
- **备选**: 复杂推理可以调用DeepSeek

### 4.2 安全第一

- **所有进化**: 用户确认后才应用
- **重大变更**: 必须验证后才能生效
- **回滚机制**: 随时可回滚到任意历史点

### 4.3 量化领域优先

- 专注优化策略、回测、优化器、风控
- 不做通用对话
- 每个技能都要验证实际量化场景效果
