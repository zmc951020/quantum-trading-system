
# QS Robot 完整架构设计 v2.0

## 机器人核心定位

**QS Robot = 系统神经中枢 + 智能助手平台**

- **第一要务 (P0)**: 系统神经中枢 - 便捷集成使用系统各模块 + 深度优化
- **第二要务 (P1)**: 6个专家系统 - 专业分析和建议（小功能）
- **第三要务 (P2)**: 技能克隆器 - 获取外部技能（小功能）

## 二、能力边界定义

### 机器人做什么（能力范围）

| 层级 | 能力类型 | 说明 |
|------|---------|------|
| **P0 - 必须** | 系统神经中枢 | 集成使用系统各模块 + 深度优化 |
| **P1 - 重要** | 6个专家系统 | 专业分析和建议 |
| **P2 - 增益** | 技能克隆器 | 获取外部技能 |

### 机器人不做什么（边界）

1. **不替换现有系统** - 保持现有系统原样，机器人作为增益层
2. **不直接执行交易** - 只提供建议，不干涉交易执行
3. **不做通用对话** - 专注量化领域，不闲聊
4. **不随意修改系统配置** - 所有操作都需要用户确认

## 三、各层详细设计

### L6 - 用户交互层

**功能**:
- Web界面（聊天窗口 + 控制面板）
- CLI命令行工具
- API接口供其他系统调用

---

### L5 - 指挥调度层（核心）

**功能**:
- NLU理解：解析用户自然语言指令
- 路由决策：决定调用哪个能力/工具
- 对话管理：维护上下文，理解多轮对话
- 工具调度：组合调用多个工具完成任务

**借鉴**: langchain-trading-agents的AiBusControl

---

### L4 - 能力层（核心）

#### P0 - 核心能力：系统神经中枢

**能力1: 系统模块集成层**

| 模块 | 功能 |
|------|------|
| 策略系统集成 | 查看策略、启动/停止、配置参数 |
| 优化器集成 | 选择优化器、配置参数、启动/监控优化 |
| 回测系统集成 | 配置回测、运行回测、查看报告 |
| 风控系统集成 | 查看风控指标、调整风控规则 |
| 实时交易集成 | 查看持仓、盈亏、订单状态 |
| 系统健康集成 | 检查系统状态、重启模块 |

**能力2: 深度优化引擎**

| 能力 | 说明 |
|------|------|
| 策略优化建议 | 分析策略表现、给出改进建议 |
| 参数优化推荐 | 智能推荐参数配置 |
| 风控优化建议 | 优化风控规则 |
| 系统性能优化 | 性能调优、配置优化 |

---

#### P1 - 增益能力：6个专家系统

| 专家 | 职责 |
|------|------|
| 策略专家 | 策略分析、推荐、诊断 |
| 优化器专家 | 优化器选择、参数建议、进度监控 |
| 回测专家 | 回测配置、报告解读、问题诊断 |
| 风控专家 | 风险检查、规则解释、防钓鱼 |
| 系统健康官 | 系统状态、日志分析、健康报告 |
| 交易监控官 | 持仓监控、盈亏统计、风险预警 |

**说明**: 6个专家只是机器人的一个小功能集，不是核心架构

---

#### P2 - 增益能力：技能克隆器

**功能**:
- 发现技能：从GitHub/市场搜索相关技能
- 克隆技能：使用create-agent-skill克隆外部技能
- 验证技能：安全扫描 + 测试运行
- 安装技能：加入机器人的技能库
- 管理技能：启用/禁用、评分、卸载

**借鉴**: create-agent-skill + ghrepo-mcp

**说明**: 技能克隆器只是机器人的另一个小功能，不是核心

---

### L3 - 工具层

**内置工具集**:
- 系统集成工具：调用各系统模块的工具
- 分析工具：数据解析、报告生成、诊断分析
- 操作工具：配置修改、参数调整、优化启动

**外部技能集**:
- 克隆的外部技能（P2功能）

---

### L2 - 记忆层（5层）

**借鉴**: GenericAgent的5层记忆系统

| 层级 | 名称 | 作用 |
|------|------|------|
| L0 | Meta Rules | 基础行为约束（宪法） |
| L1 | Insight Index | 快速路由索引 |
| L2 | Global Facts | 长期稳定知识 |
| L3 | Task Skills | 可复用任务流程 |
| L4 | Session Archive | 会话归档记录 |

---

### L1 - 基础设施层

**LLM接口层**:
- 优先：Ollama本地模型（qwen2.5-coder）
- 备选：DeepSeek API（复杂推理）

**监控钩子层**:
- Pre-Tool Hook：工具调用前监控
- Post-Tool Hook：工具调用后监控
- 信号检测：检测学习信号

**借鉴**: Self-Improving Agent的钩子机制

## 五、关键设计原则

1. **系统神经中枢第一 (P0)**: 集成使用系统各模块 + 深度优化
2. **不替换现有系统**: 保持现有系统原样，机器人作为增益层
3. **便捷性**: 简化复杂操作，降低使用门槛
4. **深度**: 不仅是操作，还要有深度的优化建议
5. **安全**: 所有敏感操作都需要用户确认
6. **本地优先**: 优先使用Ollama本地模型，保护隐私
