
# QS Robot 完整架构

## 核心定位

**我们的系统已经很先进了，机器人作为增益层，不是来替换它的！**

- 核心能力1：**6个专家系统** (策略、优化器、回测、风控、健康官、监控官)
- 核心能力2：**技能克隆器** (爬取、获取、吸收外部Skills)

## 6个专家系统设计

### 1. 策略专家 (StrategyExpert)

**能力**:
- 解释策略逻辑
- 分析策略表现
- 推荐策略组合
- 策略参数建议

**不做**: 替换策略逻辑本身

---

### 2. 优化器专家 (OptimizerExpert)

**能力**:
- 选择合适的优化器
- 配置优化参数
- 监控优化进度
- 解读优化结果

**不做**: 替换优化算法

---

### 3. 回测专家 (BacktestExpert)

**能力**:
- 配置回测参数
- 运行回测
- 解读回测报告
- 找改进点
- 对比回测结果

**不做**: 替换回测引擎

---

### 4. 风控专家 (RiskExpert)

**能力**:
- 检查风险指标
- 解释风控规则
- 防钓鱼监控
- 风险预警建议

**不做**: 替换风控逻辑

---

### 5. 系统健康官 (HealthOfficer)

**能力**:
- 系统健康检查
- 日志分析
- 性能监控
- 问题诊断
- 维护建议

**不做**: 系统运维操作

---

### 6. 交易监控官 (MonitorOfficer)

**能力**:
- 实时持仓监控
- 盈亏统计
- 策略状态监控
- 异常预警

**不做**: 交易执行

## 技能克隆器 (SkillCloner) 设计

### 工作流程

```
用户: "克隆一个数据可视化技能"
  ↓
1. 发现 (crawl): 在GitHub/市场搜索
2. 拉取 (fetch): clone仓库
3. 分析 (analyze): 用ghrepo-mcp分析
4. 转换 (transform): 用create-agent-skill转换
5. 验证 (verify): 安全扫描 + 测试
6. 安装 (install): 放入.qs_robot/skills/external/
7. 注册 (register): 添加到索引
```

### 数据来源

- GitHub 直接仓库
- Claude Plugin Hub
- 其他技能市场
- 我们自己编写的内部技能

## 界面命令示例

```
# 专家相关
/strategy help                # 策略专家帮助
/optimizer suggest            # 优化器专家建议
/backtest analyze             # 回测专家分析
/risk check                   # 风控专家检查
/health status                # 健康官检查
/monitor status               # 监控官状态

# 技能克隆相关
/skills list                  # 列出已安装技能
/skills search "可视化"       # 搜索技能
/skills clone &lt;github-url&gt;    # 克隆技能
/skills install &lt;skill&gt;       # 安装技能
/skills disable &lt;skill&gt;       # 禁用技能
```
