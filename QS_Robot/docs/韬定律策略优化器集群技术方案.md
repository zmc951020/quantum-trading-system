# 韬定律策略优化器集群技术方案

**项目名称**: OptimizerTauCluster（韬定律策略优化器集群）

**版本**: v1.0

**日期**: 2026-06-03

**设计理念**: 借鉴华为"韬(τ)定律"核心思想——以"时间缩微"替代"空间堆砌"，通过全栈协同优化实现策略优化效率的量级提升。

---

## 一、设计背景与目标

### 1.1 现状问题

当前量化交易系统的策略优化存在以下问题：

| 问题 | 描述 | 影响 |
|------|------|------|
| 串行优化 | 6个优化器依次运行，一个完成再运行下一个 | 时间浪费严重 |
| 重复计算 | 各优化器独立搜索，相同参数组合被重复计算 | 计算资源浪费 |
| 空间重叠 | 各优化器搜索同一参数空间，缺乏协同 | 效率低下 |
| 策略-优化器错配 | 不同类型策略使用相同优化器 | 收敛慢、易陷入局部最优 |
| 结果孤岛 | 各优化器结果不共享，无法相互借鉴 | 错过跨优化器的有价值发现 |

### 1.2 设计目标

```
τ_优化 = (参数空间维度 × 单次回测耗时) / 有效搜索效率

目标: 最小化 τ_优化，实现 3-5倍效率提升
```

**缓存键设计**:
```python
def make_cache_key(strategy_name: str, params: Dict,
                   start_date: str, end_date: str) -> str:
    """生成缓存键"""
    # 1. 参数标准化 (忽略微小子差异)
    normalized = normalize_params(params)
    # 2. 组合特征
    features = [strategy_name, start_date, end_date,
                hash_dict(normalized)]
    return "|".join(features)
```

**淘汰策略**: LRU + 命中率阈值，保留命中率 > 10% 的缓存

### 3.2 电路层: ParameterSpaceFolding

**职责**: 参数空间折叠，将高维空间分解为多层子空间

```python
class ParameterSpaceFolding:
    """
    参数空间折叠层 - τ定律"电路层"实现

    核心思想: 不是让6个优化器同时搜索同一空间，
    而是让每个优化器专注于空间的特定"层级"。
    """

    class SpaceLevel(Enum):
        COARSE = "coarse"      # 粗筛层
        REFINED = "refined"    # 精搜层
        VALIDATION = "validation"  # 验证层

    def __init__(self, param_space: Dict[str, ParamSpec]):
        """
        Args:
            param_space: 参数空间定义
                {
                    "period_short": (5, 200, 1),   # (min, max, step)
                    "period_long": (10, 500, 1),
                    "threshold": (0.01, 0.1, 0.001),
                    ...
                }
        """
        self.raw_space = param_space
        self.subspaces = self._create_folded_spaces()

    def _create_folded_spaces(self) -> Dict[SpaceLevel, List[SubSpace]]:
        """
        创建折叠后的子空间

        粗筛层: 稀疏网格 (step放大10倍)
        精搜层: 稠密网格 (聚焦粗筛高分区域)
        验证层: 原始精度 (最终验证)
        """
        subspaces = {}

        # 粗筛空间: 0.1% ~ 1% 的原始空间
        subspaces[SpaceLevel.COARSE] = self._make_coarse_grid()

        # 精搜空间: 粗筛Top20%区域 × 10倍精度
        subspaces[SpaceLevel.REFINED] = self._make_refined_grid(
            top_regions=self._find_top_regions(k=0.2)
        )

        # 验证空间: 精搜Top10区域的原始精度版本
        subspaces[SpaceLevel.VALIDATION] = self._make_validation_grid(
            top_regions=self._find_top_regions(k=0.1)
        )

        return subspaces

    def get_subspace_for_optimizer(self, optimizer_type: str) -> SubSpace:
        """根据优化器类型返回对应的子空间"""
        SPACE_OPTIMIZER_MAP = {
            "grid_search": SpaceLevel.COARSE,
            "bayesian": SpaceLevel.REFINED,
            "genetic": SpaceLevel.REFINED,
            "rl_optimizer": SpaceLevel.VALIDATION,
            "enhanced": SpaceLevel.REFINED,
            "five_line": SpaceLevel.VALIDATION,
        }
        level = SPACE_OPTIMIZER_MAP.get(optimizer_type, SpaceLevel.REFINED)
        return self.subspaces[level]
```

### 3.3 芯片层: StrategyOptimizerMatcher

**职责**: 策略类型与优化器的智能匹配

```python
class StrategyOptimizerMatcher:
    """
    策略-优化器匹配器 - τ定律"芯片层"实现

    核心思想: 不同策略类型有不同的参数特征，
    应匹配最适合的优化算法。
    """

    # 策略类型 → 优化器优先级列表
    STRATEGY_OPTIMIZER_MAP = {
        "趋势跟踪": [
            ("bayesian", 1),      # 连续参数最优化
            ("grid_search", 2),   # 网格搜索兜底
            ("genetic", 3),       # 进化探索
        ],
        "均值回归": [
            ("genetic", 1),       # 多参数组合优势
            ("bayesian", 2),      # 精细调优
            ("enhanced", 3),      # 综合优化
        ],
        "网格交易": [
            ("grid_search", 1),   # 离散参数天然适合
            ("enhanced", 2),     # 参数多时用增强版
            ("bayesian", 3),
        ],
        "机器学习预测": [
            ("rl_optimizer", 1),  # 强化学习适配动态环境
            ("genetic", 2),       # 进化策略
            ("bayesian", 3),
        ],
        "多因子共振": [
            ("enhanced", 1),      # 综合调优
            ("bayesian", 2),
            ("five_line", 3),     # 五行门系统
        ],
        "陀螺仪策略": [
            ("five_line", 1),     # 专用五行逻辑
            ("genetic", 2),
            ("rl_optimizer", 3),
        ],
        "伯努利-康达": [
            ("bayesian", 1),
            ("genetic", 2),
            ("enhanced", 3),
        ],
        "Shepherd系列": [
            ("genetic", 1),
            ("five_line", 2),
            ("rl_optimizer", 3),
        ],
        "默认": [
            ("enhanced", 1),
            ("bayesian", 2),
            ("genetic", 3),
        ]
    }

    def match(self, strategy_type: str,
              available_optimizers: List[str]) -> List[Tuple[str, float]]:
        """
        返回匹配度排序的优化器列表

        Returns:
            [(optimizer_name, weight), ...] 按优先级排序
        """
        mapping = self.STRATEGY_OPTIMIZER_MAP.get(
            strategy_type,
            self.STRATEGY_OPTIMIZER_MAP["默认"]
        )

        # 只返回系统中可用的优化器
        available_set = set(available_optimizers)
        result = [(opt, weight) for opt, weight in mapping
                  if opt in available_set]

        return result if result else [("enhanced", 1.0)]

    def get_adaptive_weights(self, strategy_type: str,
                            intermediate_results: Dict[str, OptimizerResult]
                            ) -> Dict[str, float]:
        """
        根据中间结果动态调整优化器权重

        Args:
            intermediate_results: 各优化器当前最优结果

        Returns:
            {optimizer_name: weight} 权重会随表现动态变化
        """
        base_weights = dict(self.match(strategy_type, ...))

        # 表现好的优化器权重上调
        for opt_name, result in intermediate_results.items():
            if result.score > baseline:
                base_weights[opt_name] *= (1 + improvement_factor)

        # 归一化
        total = sum(base_weights.values())
        return {k: v/total for k, v in base_weights.items()}
```

### 3.4 系统层: OptimizerBus + ResourceScheduler

#### OptimizerBus: 优化器间通信总线

```python
class OptimizerBus:
    """
    优化器结果共享总线 - τ定律"系统层"实现

    核心思想: 借鉴"灵衢总线"的统一内存编址理念，
    各优化器的发现（热点区域、坏区域）应被共享。
    """

    def __init__(self):
        # 全局热点区域 (各优化器发现的高价值区域)
        self.hot_regions: List[Region] = []
        # 全局排除区域 (各优化器验证为差劲的区域)
        self.excluded_regions: List[Region] = []
        # 订阅者列表
        self._subscribers: List[Optimizer] = []

    def subscribe(self, optimizer: Optimizer):
        """优化器订阅总线"""

    def publish(self, optimizer_name: str, findings: List[Finding]):
        """
        发布优化器发现

        Finding类型:
            - HotRegion: 高潜力区域
            - ExcludedRegion: 已排除区域
            - NewPattern: 新发现模式
        """
        for finding in findings:
            if isinstance(finding, HotRegion):
                self.hot_regions.append(finding)
                self._notify_subscribers(finding)
            elif isinstance(finding, ExcludedRegion):
                self.excluded_regions.append(finding)
                self._notify_subscribers(finding)

    def get_guidance(self, optimizer_name: str) -> SearchGuidance:
        """
        获取对特定优化器的搜索指导

        指导内容:
            - 热点区域列表 (鼓励探索)
            - 排除区域列表 (避免重复)
            - 推荐子空间 (根据策略类型)
        """
        return SearchGuidance(
            encourage_regions=self.hot_regions,
            avoid_regions=self.excluded_regions,
            best_params=self._get_best_params_so_far()
        )
```

#### ResourceScheduler: 动态资源调度器

```python
class ResourceScheduler:
    """
    资源调度器 - τ定律"系统层"性能保障

    核心思想: 差的子空间应该被快速淘汰，
    优秀的表现应该获得更多资源。
    """

    def __init__(self, max_workers: int = 6):
        self.max_workers = max_workers
        self._optimizer_states: Dict[str, OptimizerState] = {}
        self._quota: Dict[str, float] = {}  # CPU配额 0.0-1.0

    def update_quota(self, optimizer_name: str, performance: float):
        """
        根据表现更新资源配额

        规则:
            - 性能高于均值: 配额上调
            - 连续K次无进展: 暂停/减少配额
            - 性能低于均值: 配额下调
        """
        state = self._optimizer_states[optimizer_name]

        if state.consecutive_no_improvement > K_THRESHOLD:
            # 暂停或大幅减少配额
            self._quota[optimizer_name] *= 0.1
        elif performance > state.avg_performance * 1.2:
            # 表现优秀，增加配额
            self._quota[optimizer_name] = min(1.0,
                self._quota[optimizer_name] * 1.5)
        else:
            # 正常配额
            pass

    def get_active_optimizers(self) -> List[str]:
        """返回当前应激活的优化器列表"""

    def should_terminate(self, optimizer_name: str) -> bool:
        """判断是否应终止某个优化器"""
```

## 四、核心流程

### 4.1 优化集群启动流程

```
用户: "优化策略 趋势跟踪策略"

1. 策略类型识别
   StrategyTypeDetector → "趋势跟踪"

2. 参数空间生成
   ParameterSpaceFolding → 粗筛/精搜/验证三层空间

3. 优化器匹配
   StrategyOptimizerMatcher → [(bayesian, 0.5), (grid, 0.3), ...]

4. 集群启动
   OptimizerTauCluster.run():
       │
       ├── ThreadPoolExecutor 启动 6 个优化器
       │
       ├── OptimizerBus.start() 开启结果共享
       │
       ├── ResourceScheduler.start() 开启动态调度
       │
       └── BacktestCache.warm_start() 预热缓存

5. 迭代优化 (每 N 秒同步一次)
   for iteration in range(max_iterations):
       │
       ├── 各优化器在分配的子空间搜索
       │
       ├── OptimizerBus.sync() 同步中间结果
       │
       ├── ResourceScheduler.adjust() 调整配额
       │
       └── BacktestCache.update() 更新缓存

6. 结果聚合
   MultiObjectiveAggregator.aggregate()
   │
   ├── 多目标评分
   ├── 交叉验证
   └── 鲁棒性检查

7. 返回最优参数
```

## 七、风险与应对

| 风险 | 描述 | 应对 |
|------|------|------|
| 空间折叠丢失最优解 | 粗筛可能错误淘汰潜在好区域 | 保留少量"探索配额"不被折叠限制 |
| 优化器间竞争资源 | 动态调度可能饿死某些优化器 | 设置最小配额保障 |
| 缓存一致性 | 并发写入可能破坏缓存 | 使用线程安全的数据结构 |
| 结果冲突 | 不同优化器给出矛盾结论 | 多目标聚合器做最终裁决 |
